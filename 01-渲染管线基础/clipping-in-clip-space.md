# Q01.03 什么是裁剪（Clipping）？为什么它通常在齐次裁剪空间（Clip Space）中进行？

**难度：** 🟢 初级
**所属章节：** [渲染管线基础](./index.md)

---

## 🎯 结论

- 裁剪是将视锥体（View Frustum）外的图元部分或全部剔除的过程，避免处理不可见的几何体。
- 在齐次裁剪空间（Clip Space）中进行裁剪，是因为透视投影的除法（透视除法）尚未执行，可以统一处理所有图元类型。
- 裁剪后进行透视除法得到NDC坐标，再映射到屏幕空间。

## 📐 原理解析

### 裁剪空间（Clip Space）的几何意义

- 顶点经过MVP变换后进入裁剪空间，此时坐标为齐次坐标(x, y, z, w)
- w分量的物理意义：在透视投影中，w = -z_view（观察空间深度的负值），代表顶点到相机的距离
- 裁剪条件：-w <= x <= w, -w <= y <= w, 0 <= z <= w（OpenGL中z范围为[-w, w]）
- 满足所有条件的点在视锥体内部，任一条件不满足则该点在视锥体外部

### 为什么在Clip Space裁剪

- 透视除法（x/w, y/w, z/w）在w=0时未定义（相机位置处的点），必须在除法之前裁剪
- 在Clip Space中，裁剪条件对所有投影类型（透视/正交）统一为 -w <= x,y,z <= w 的形式
- 正交投影中w恒为1，裁剪退化为 -1 <= x,y,z <= 1（即NDC范围），与透视投影用同一套逻辑处理
- 在观察空间或世界空间裁剪需要处理6个不同方程的平面，而Clip Space只需检查w的关系

### 裁剪算法

- 硬件通常使用Sutherland-Hodgman算法的变体：逐边裁剪多边形
- 对于三角形：最多产生7个顶点（3个顶点 + 最多4个新增交点）
- 裁剪后的三角形可能被分解为三角形扇（Triangle Fan）或多个三角形送入后续管线


## 🛠 工程实践

- 自定义裁剪平面的实现：通过gl_ClipDistance[]（GLSL）或SV_ClipDistance（HLSL）设置用户裁剪平面。在VS中计算顶点到平面的有符号距离，硬件自动裁剪。
- 用户裁剪平面的典型应用场景：水面裁剪（渲染水下场景时裁剪水面以上部分）、Portal效果（门/窗口限制渲染区域）、阴影级联边界裁剪。
- 在Shader中启用裁剪平面时需要注意性能影响：额外的裁剪平面会增加硬件裁剪器的工作量，通常限制在4-6个以内。
- Unity中通过Camera.layerCullDistances和Shader中的clip()函数可以实现自定义裁剪逻辑。

## ⚠️ 踩坑经验

- 近平面裁剪导致物体消失：当物体非常靠近相机时，部分三角形可能完全在近平面之前被裁剪掉，导致物体出现"空洞"。解决方案是减小近平面距离或使用无限远投影矩阵。
- 大尺度场景的z-fighting：当远近平面比值（Far/Near）过大时，深度缓冲精度不足，导致远处物体出现闪烁。解决方案包括对数深度缓冲（Logarithmic Depth Buffer）和反转深度缓冲（Reversed-Z）。
- 裁剪平面的法线方向容易搞反：gl_ClipDistance > 0表示在平面内侧（保留），< 0表示在外侧（裁剪），需要仔细确认。
- 移动端某些GPU对用户裁剪平面的支持有限，需要在目标设备上测试验证。

## 🔮 延伸思考

- 无限远投影矩阵（Infinite Projection Matrix）通过将远平面推向无穷远来消除远平面裁剪，同时保持近平面裁剪。这在渲染天空盒和远距离场景时非常有用，且不损失近处的深度精度。
- 反转深度缓冲（Reversed-Z）将深度范围从[0,1]反转为[1,0]，配合浮点Depth Buffer可以将精度分布从"近处密远处疏"变为"近处密远处也密"，大幅改善z-fighting问题。
- Clustered Rendering中，裁剪空间的概念被扩展到3D空间：将视锥体沿深度方向切分为多个Cluster，每个Cluster独立处理光源归属，避免了传统Tiled Forward在深度方向上的精度问题。
