# Q03.06 在大型开放世界中，如何设计一个高效的阴影系统？请讨论CSM与距离场阴影的配合策略。

**难度：** 🔵 高级
**所属章节：** [阴影技术](./index.md)

---

## 🎯 结论

- 大型开放世界的阴影系统设计是渲染工程中最具挑战性的任务之一，需要同时处理超大范围的阴影覆盖、动态物体的实时阴影、以及性能预算的严格限制。
- 业界主流的分层策略是：近处使用CSM提供高精度实时阴影，中远距离使用距离场阴影提供软阴影和环境遮蔽，静态物体使用预烘焙阴影。这种分层方案在视觉质量和性能之间取得了良好的平衡。

## 📐 原理解析

### 距离场阴影的基本原理

- 距离场（Signed Distance Field, SDF）存储空间中每个点到最近表面的距离（内部为负，外部为正）。对于阴影计算，我们主要关注外部距离场。
- 距离场阴影通过光线步进（Ray Marching）来计算：从接收面表面沿光线方向步进，如果在到达光源之前遇到距离场值接近0的区域，则说明存在遮挡物。软阴影的半影宽度可以通过在步进过程中累积遮挡量来估算——距离场值越小表示越接近表面，遮挡贡献越大。
- 距离场通常在离线阶段预计算，存储为3D体素纹理。对于静态场景，距离场只需要计算一次；对于动态物体，需要实时更新其距离场，这是一个计算密集的操作。

### UE中的距离场阴影实现

- UE使用全局距离场（Global Distance Field）将场景的距离场信息合并为一个低分辨率的3D纹理，覆盖整个场景。每个体素存储到最近表面的距离。
- 距离场阴影在UE中主要用于提供大范围的软阴影和环境遮蔽（Contact Shadows的补充）。它特别适合静态建筑、地形等大型物体的阴影投射。
- UE的DF阴影通过在屏幕空间进行光线步进来实现，利用Global Distance Field的Clipmap结构来支持超大范围的查询。


## 🛠 工程实践

### 分层阴影策略的具体配置

- 第一层（0-15m）：CSM Cascade 0，最高分辨率（如2048x2048），处理玩家附近的高精度实时阴影，包括角色、近处道具和地面细节。
- 第二层（15-50m）：CSM Cascade 1-2，中等分辨率（如1024x1024或2048x2048），覆盖中距离的建筑和植被阴影。
- 第三层（50-200m）：距离场阴影，提供大范围的软阴影。使用Global Distance Field的Clipmap，分辨率随距离递减。
- 第四层（200m+）：预烘焙的Lightmap和Ambient Occlusion，提供静态环境的光照和遮蔽信息。
- 动态物体（如移动的NPC、车辆）：在CSM范围内通过实时Shadow Map投射阴影；超出CSM范围的动态物体使用屏幕空间Contact Shadows或简单的胶囊体阴影近似。

### 动态物体的距离场更新策略

- 对于少量关键动态物体（如玩家角色、主要NPC），可以为其生成局部的距离场并注入Global Distance Field。这需要在每帧或每隔几帧重新计算受影响区域的距离场。
- UE使用Object Distance Field为每个动态物体预计算其距离场（在物体加载时），然后在运行时将其注入到Global Distance Field的对应位置。这种方法的开销与动态物体的数量和大小相关。
- 对于大量小型动态物体（如碎片、植被），通常不使用距离场阴影，而是使用CSM或简化的阴影近似（如Blob Shadows）。


## ⚠️ 踩坑经验

### 距离场阴影的分辨率与内存平衡

Global Distance Field的内存开销与场景大小和分辨率直接相关。一个覆盖1km x 1km x 200m的场景，如果体素分辨率为0.5m，需要约8GB的内存（仅存储距离值），这显然不可接受。实践中通常使用非常低的分辨率（如2-4m体素），并依赖Clipmap结构在近处提供较高精度。即使如此，大型开放世界的Global Distance Field仍可能消耗数百MB的内存，需要在视觉质量和内存预算之间仔细权衡。

### 距离场生成时间对关卡加载的影响

- 距离场的计算是计算密集型的离线操作。对于大型场景，距离场生成可能需要数分钟到数十分钟，严重影响关卡迭代速度。
- UE采用增量计算和异步生成策略来缓解这个问题。在编辑器中修改场景后，只重新计算受影响区域的距离场。但在首次加载或大规模修改后，仍需要较长的等待时间。
- 运行时的距离场更新（用于动态物体）需要严格控制计算预算，通常使用Compute Shader并行化处理，并限制每帧的更新量。


## 🔮 延伸思考

### Nanite+Lumen时代距离场阴影的前景

UE5的Nanite虚拟几何体和Lumen全局光照系统正在改变渲染管线的方方面面。Lumen使用光线追踪（软件或硬件）来计算间接光照和反射，同时也提供直接光照的软阴影。在Lumen管线中，距离场阴影的作用正在被重新评估——Lumen的Ray Tracing可以直接提供高质量的软阴影，无需预计算距离场。然而，距离场在Lumen中仍被用作光线追踪的加速结构（用于快速求交），因此距离场本身可能不会被淘汰，但其作为独立阴影技术的角色可能会逐渐弱化。

### 完全动态阴影的未来

随着硬件光线追踪的普及和算法的进步，完全动态的阴影系统正在成为可能。Ray-Traced Shadows可以自然地处理任意几何形状的阴影投射，支持面积光源的物理正确软阴影，且不需要预计算或级联分割。当前的主要瓶颈在于性能——尤其在移动端和大规模场景中。未来的发展方向包括：基于AI的超分辨率和去噪技术来降低光线追踪的采样需求、混合光线追踪方案（近处RT+远处CSM/DF）、以及专用的光线追踪硬件加速。
